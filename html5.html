sessionStorage和localStorage

sessionStorage用于本地存储一个会话（session）中的数据，这些数据只有在同一个会话中的页面才能访问并且当会话结束后数据也随之销毁。

而localStorage用于持久化的本地存储，除非主动删除数据，否则数据是永远不会过期的。

localStorage和sessionStorage都具有相同的操作方法，例如setItem、getItem和removeItem等

sessionStorage.setItem("key", "value");\\存储
sessionStorage.key = 'value';\\存储
localStorage.getItem("site");\\获取
localStorage.site;\\获取
sessionStorage.removeItem("key");\\删除
localStorage.clear();\\清除

HTML5离线存储

HTML5提供了离线存储方案，顾名思义就是在离线的情况下访问网站资源。嗯~你也许会想到90年代的脱机浏览和离线软件！

原理

原理也比较简单，就是根据manifest离线配置文件对相应的资源进行存储，下次访问时直接从cache中读取

怎么使用？

AddType text/cache-manifest manifest
AddType text/cache-manifest .appcache
<html manifest="cache.manifest">

HTML5离线存储——manifest文件

CACHE MANIFEST
# 版本号信息
# version 1.2 2012-6-21
# 需缓存的文件(注意当前页面是默认缓存的)
CACHE:
css/reset.css
../html5/audio/zxmzf.ogg
# 不需要缓存的文件，一直访问在线的文件 [可选]
# ps：如果不加 * 默认会请求不到内容
NETWORK:
*
# 当离线访问未缓存资源时的替代方案 [可选]
#FALLBACK:
/index.html /offline.html
/ /offline.html

HTML5离线存储——chrome console

HTML5离线存储——检查缓存状态

window.applicationCache.status

数 字 状 态 描 述
0 uncached 页面不在应用程序缓存中。应用程序缓存第一次加载时页面也处于这种状态
1 idle 当应用程序缓存是最新的时，浏览器将状态设置为idle
2 checking 当应用程序检查是否有更新了的清单文件时，浏览器将状态设置为checking
3 downloading 当应用程序正在下载新缓存时，浏览器将状态设置为downloading
4 updateready 当新缓存已下载完毕，可以替换旧有资源时，浏览器将状态设置为updateready
5 obsolete 当找不到清单文件时，浏览器将状态设置为obsolete

上面对应各自事件，例如：window.applicationCache.onupdateready

WebSocket

WebSockets是在一个(TCP)接口进行双向通信的技术，PUSH技术类型。同时WebSockets仍将基于W3C标准，目前为止，Chrome和Safari的最新版本浏览器已经支持WebSockets了。

优势

  • 服务器与客户端之间交换的标头信息很小，大概只有2字节
  • 服务器可以主动传送数据给客户端
  • 客户端开发接口简单

    过渡：transition

    transition是指过渡动画，让css属性的变化是可连续的、可控时间的、可控变化曲线的做出改变，无论是鼠标事件还是点击事件，只要设置的CSS属性发生改变，transition就能让元素在设置的时间内平滑的发生改变

    transition四个子属性

    • property：需要变化的值(属性)
    • duration：持续时间，单位s、ms
    • timing-function：ease,ease-in,ease-out,ease-in-out和cubic-bezier贝塞尔曲线
    • delay：延时，从多少时间后开始，默认为0

    转换：transform

    transform指转换(变换)。transform包括：倾斜(skew)，缩放(scale)，旋转(rotate)，位移(translate)和矩阵(matrix)

    • skew：倾斜转换
    • scale：缩放比例，如1.5
    • rotate：旋转角度，30deg
    • translate：位移
    • matrix：变换矩阵，6个数字组成的矩阵
    • transform-origin：变换基点
